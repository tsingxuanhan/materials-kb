## 一、LLM-Native Cognitive Architectures

### 1. Letta (原 MemGPT)
- **GitHub**: https://github.com/letta-ai/letta
- **Stars**: ~22,400 (letta) + ~30K (原始 cpacker/MemGPT)
- **最近更新**: 2026-03-31 (v0.16.7)
- **核心能力**: OS式虚拟上下文管理——将LLM上下文窗口视为RAM，外部存储视为硬盘，Agent自主决定信息在三级存储间的调度
- **技术栈**: Python, Apache 2.0, pip install letta-client; 支持pgvector/SQLite/Pinecone
- **RAM/VRAM**: 服务端2-4GB RAM；本地嵌入模型额外需~2GB
- **集成路径**: **独立服务** — Letta是完整的Agent运行时，不是可插拔的记忆层。集成方式是将xuanshu-agents的Agent注册为Letta Agent类型，通过Letta Server API交互
- **集成难度**: 4/5 — 需要接受Letta的Agent运行时范式，记忆系统与现有n-gram TF-IDF存在架构冲突

### 2. Agent Zero
- **GitHub**: https://github.com/agent0ai/agent-zero
- **Stars**: ~13-15K
- **最近更新**: 持续活跃 (2026)
- **核心能力**: 全Prompt驱动的自主Agent框架——OS即工具，Agent可自主编写代码创建新工具，FAISS向量记忆+层级Agent系统
- **技术栈**: Python 3.8+, Docker, FAISS, 支持OpenAI/Anthropic/Groq/Gemini/Ollama
- **RAM/VRAM**: Docker容器运行，基础~4GB RAM；本地模型另计
- **集成路径**: **插件式** — Agent Zero的7柱架构（Agents/Tools/Memory/Prompts/Knowledge/Skills/Extensions）可通过Extension系统注入xuanshu-agents的Agent行为
- **集成难度**: 3/5 — Extension点定义清晰，可单独提取其FAISS记忆或层级委派机制

### 3. NeuroCortex
- **GitHub**: https://github.com/Ringlingo/NeuroCortex
- **Stars**: 新项目 (<1K)
- **最近更新**: 2026-05
- **核心能力**: 零依赖纯文本LLM认知协议——用Markdown文件构建可自我进化的记忆与规则系统，含L3层认知架构(知识/规则/经验)和域冲突裁决
- **技术栈**: 零依赖，纯Markdown/文本文件
- **RAM/VRAM**: 几乎为零（纯文本+LLM API调用）
- **集成路径**: **直接嵌入** — 协议层面的设计可直接映射到xuanshu-agents的memory_provider体系中，INTUITION.md + EPISODES.md可作为新的memory provider
- **集成难度**: 2/5 — 零依赖，纯协议设计，可渐进式引入

### 4. Mnemosyne
- **GitHub**: https://github.com/rand/mnemosyne (推测)
- **Stars**: 中等规模
- **最近更新**: 2025-11
- **核心能力**: Rust实现的高性能Agent记忆系统——4-Agent框架(Orchestrator/Optimizer/Reviewer/Executor) + 混合搜索(FTS5+图遍历+向量) + MCP协议 + PyO3 Python绑定
- **技术栈**: Rust核心 + Python绑定(PyO3), MCP over JSON-RPC 2.0
- **RAM/VRAM**: Rust核心极高效，<500MB RAM
- **集成路径**: **独立服务** — 通过MCP协议或PyO3绑定接入，10-20x速度优势
- **集成难度**: 3/5 — MCP协议标准接口，但Rust核心需要编译部署

## 二、Memory-Augmented Cognition

### 5. Mem0
- **GitHub**: https://github.com/mem0ai/mem0
- **Stars**: ~52,500
- **最近更新**: 持续活跃 (2026)
- **核心能力**: 第三代"记忆即基础设施"——多引擎(向量+图+全文+结构化)统一记忆层，$24M融资，支持MCP协议
- **技术栈**: Python, 向量DB(pgvector/Qdrant), 图DB(Neo4j), pip install mem0ai
- **RAM/VRAM**: 云端部署为主；自托管~4-8GB
- **集成路径**: **独立服务/MCP** — Mem0已支持MCP协议，可作为xuanshu-agents的记忆后端服务
- **集成难度**: 3/5 — MCP标准接口，但需要替换现有n-gram方案或并行运行

### 6. MemCore
- **GitHub**: PyPI: memcore-ai
- **Stars**: 新项目
- **最近更新**: 2026-03
- **核心能力**: 轻量级三层记忆系统——Core(工作记忆,~2000 tokens) + Recall(情景记忆) + Archival(语义记忆)，直接受MemGPT启发
- **技术栈**: Python 3.8+, MIT, pip install memcore-ai; 可选向量搜索
- **RAM/VRAM**: 极轻量，<500MB
- **集成路径**: **直接嵌入** — 纯Python库，可直接替换xuanshu-agents的SlotMemoryProvider
- **集成难度**: 2/5 — API简洁，三层架构与xuanshu-agents现有记忆体系高度兼容

### 7. Hindsight
- **GitHub**: 独立项目 (~4K stars)
- **Stars**: ~4K
- **最近更新**: 2026
- **核心能力**: 独立记忆层——4路并行检索(语义+关键词+图谱+时序) + Cross-Encoder重排序，LongMemEval基准91.4%
- **技术栈**: Python/TypeScript/Go SDK, MCP-native, Docker部署
- **RAM/VRAM**: Docker自托管~2-4GB
- **集成路径**: **插件式** — 作为记忆后端层，与任何Agent框架兼容，xuanshu-agents可将其作为retrieval provider
- **集成难度**: 2/5 — 纯记忆层，无框架绑定，MCP原生

### 8. LangMem + LangGraph Store (MongoDB)
- **GitHub**: langchain-ai/langmem + langgraph-store-mongodb
- **Stars**: LangChain生态 ~128K总星
- **最近更新**: 2025-08
- **核心能力**: 预构建记忆工具集——提取和管理procedural/episodic/semantic三类记忆，与MongoDB Atlas向量搜索集成
- **技术栈**: Python, MongoDB Atlas, pip install langgraph-store-mongodb
- **RAM/VRAM**: 云端MongoDB，本地<1GB
- **集成路径**: **插件式** — 可提取其记忆提取逻辑作为独立模块
- **集成难度**: 3/5 — 依赖LangChain生态，但记忆类型划分思路可独立借鉴

## 三、Self-Improving Agents

### 9. Hermes Agent
- **GitHub**: https://github.com/NousResearch/hermes-agent

- **RAM/VRAM**: 基础~2-4GB RAM；无本地模型时<1GB
- **集成路径**: **直接嵌入** — SKILL.md格式已是开放标准，其self-evolution机制(DSPy+GEPA)可直接为xuanshu-agents提供技能优化管道
- **集成难度**: 2/5 — SKILL.md格式完全兼容，技能自创建/自修复机制可直接引入

## 集成路线图建议

### Phase 1: 零成本嵌入 (1-2周)
- **Agent Reasoning** 代理层 → 为4个Agent增加推理策略
- **Reflexion** 模式 → 嵌入矿工/试金的review循环
- **Agent Patterns Self-Critique** → 嵌入铸师的质量门控
